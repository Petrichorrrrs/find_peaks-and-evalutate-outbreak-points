# 预警性能评估模块说明

## 一、模块定位

评估模块是整个系统的 **最后一环**，独立于上游算法（CNE / tRCE / Fusion），专门负责检验预警效果。

```
上游算法 → 预警点索引 → 评估模块 → FAR / ALT / EWR + 评估结果图
```

评估模块不做预警，只做**检验**——用真实 ILI 暴发峰作为 ground truth，计算每个预警源的真报/误报/提前期。

---

## 二、核心概念

| 概念 | 说明 |
|------|------|
| **暴发峰 (Outbreak Peak)** | ILI 病例数据的真实峰值，作为评估的 ground truth |
| **预警点 (Alarm Point)** | 上游算法输出的预警时间点索引 |
| **真报 (True Alarm)** | 预警点在其后评估窗口内检测到疫情信号 |
| **误报 (False Alarm)** | 预警点在其后评估窗口内未检测到疫情信号 |
| **未评估 (Unevaluated)** | 预警点离序列末尾太近，评估窗口越界无法判断 |

### 评估指标

| 指标 | 全称 | 说明 |
|------|------|------|
| **FAR** | False Alarm Rate | 误报率 = 误报数 / (误报+真报)，越低越好 |
| **ALT** | Average Lead Time | 平均提前期（天），预警比疫情爆发提前的天数 |
| **EWR** | Early Warning Rate | 提前成功率 = 被成功预警的疫情数 / 总疫情数 |

---

## 三、模块结构

### 文件一览

| 文件 | 作用 |
|------|------|
| `evaluate_alerts.py` | 评估模块核心，提供 `AlertEvaluator` 类 |
| `evaluation.py` | 底层评估算法，提供 `WarningEvaluator` 类 |
| `find_peaks.py` | 暴发峰自动检测算法 |
| `run_evaluation.py` | **独立评估入口脚本** |

### 依赖关系

```
evaluate_alerts.py
  ├── evaluation.py        (WarningEvaluator — 逐点评估逻辑)
  ├── find_peaks.py        (detect_ili_peaks — 暴发峰检测)
  └── data_processor.py    (load_and_process_data — 数据加载)
```

## 四、接口说明

### 标准流程（三步）

```python
from evaluate_alerts import AlertEvaluator

# 1. 创建评估器
evaluator = AlertEvaluator(
    ili_raw=ili_data,
    date_list=dates,
    data_path="测试数据全国_填充.xlsx",
)

# 2. 注册预警源
evaluator.add_source("CNE",              [50, 64, 288, ...])
evaluator.add_source("tRCE",             [50, 64, 292, ...])
evaluator.add_source("Fusion(严重)",      [55, 265, 292, ...])

# 3. 执行评估（弹图确认峰值 → 评估 → 返回结果）
results = evaluator.run()

# 可选：打印对比表 + 绘制结果图
evaluator.print_comparison()
evaluator.plot_results(save_path="result.png")
```

### AlertEvaluator API

| 方法 | 说明 |
|------|------|
| `__init__(ili_raw, date_list, data_path, ...)` | 接收 ILI 数据、日期、文件路径、评估参数 |
| `add_source(name, alert_indices)` | 注册一个预警源（名称 + 预警点索引列表） |
| `run(skip_confirm=False)` | 执行完整流程：检测峰值 → 用户确认 → 逐源评估 |
| `set_outbreak_points(points)` | 直接设置暴发峰（跳过交互） |
| `print_comparison()` | 打印 FAR / ALT / EWR 对比表 |
| `plot_results(save_path)` | 绘制评估结果图，区分真报/误报/未评估 |
| `get_results()` | 获取结构化评估结果 |
| `get_plot_data()` | 获取绘图数据（供外部可视化模块使用） |

### run() 执行流程

```
  run()
  ├── _detect_and_confirm_peaks()
  │     ├── detect_ili_peaks()       自动检测峰
  │     ├── plt.show()               弹图展示
  │     ├── input()                  询问用户 y/n
  │     │     ├── y  → 使用检测结果
  │     │     └── n  → 手动输入索引
  │     └── set outbreak_points
  │
  └── 对每个源:
        └── WarningEvaluator.evaluate(alert_indices)
              ├── 逐点判断 → 真报/误报/未评估
              ├── 计算 FAR / ALT / EWR
              └── 返回结构化结果
```

## 五、如何运行

### 方式 A：独立运行（推荐）

```bash
# 交互模式 — 弹图确认峰值
python run_evaluation.py

# 跳过交互 — 直接使用自动检测（批处理）
python run_evaluation.py --skip-confirm

# 指定数据文件
python run_evaluation.py --data-path 测试数据西安渭南_填充.xlsx
```

### 方式 B：作为完整 pipeline 的一步

```bash
python run_experiment.py
# Step 6/7 自动进入评估流程，同样有峰值确认
```

### 输出文件

| 文件 | 说明 |
|------|------|
| `experiment_results/evaluation_result.png` | 评估结果图 |
| `experiment_results/evaluation_metrics.json` | 结构化评估指标 |
| `experiment_results/peak_confirmation.png` | 峰值确认图（如有保存） |

## 六、结果图解读

评估结果图由多行子图组成：

```
┌─────────────────────────────────────────────────┐
│ ILI 病例趋势与暴发峰值                           │
│ ── ILI曲线   ▼暴发峰#1  ▼#2  ▼#3              │
├─────────────────────────────────────────────────┤
│ CNE                                             │
│ ● 真报  ✗ 误报  ◇ 未评估                        │
├─────────────────────────────────────────────────┤
│ tRCE                                            │
│ ● 真报  ✗ 误报                                  │
├─────────────────────────────────────────────────┤
│ Fusion(严重)                                     │
│ ● 真报  ◇ 未评估                                │
└─────────────────────────────────────────────────┘
```

- **绿色圆形 ●** = 真报（成功预警，该预警点后确实发生了疫情）
- **红色叉号 ✗** = 误报（预警了但没发生疫情）
- **灰色菱形 ◇** = 未评估（数据末尾，窗口越界）
- **红色三角 ▼** = ILI 暴发峰（Ground Truth）
- 虚线 = 暴发峰对应的时间位置

## 七、对比表示例

```
   预警源              总数   真报   误报     FAR     ALT     EWR   检出/总疫情
   ----------------------------------------------------------------------
   CNE                  13    11      2   15.4%   41.6d    100%          4/4
   tRCE                 10     9      1   10.0%   43.2d    100%          4/4
   Fusion(严重)          10     9      1   10.0%   42.2d    100%          4/4
```

- FAR 越低越好
- ALT 越高越好（提前更多天预警）
- EWR 越高越好（覆盖更多疫情）

## 八、参数调优

评估参数在 `AlertEvaluator.__init__()` 中设置：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `eval_window` | 60 | 预警点后检测窗口大小（天） |
| `r_c` | 0.02 | 增长率阈值，越高判定越严格 |
| `delta` | 10.0 | 最小增幅阈值，越高判定越严格 |
| `max_lead` | 60 | 最大提前期，超出的预警不计入成功预警 |
